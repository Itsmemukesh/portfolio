## 2. Branch Overview

This repo has three branches, each with a distinct purpose:

| Branch | Purpose | Edit manually? |
|--------|---------|----------------|
| `main` | Source code — where all your edits live | Yes |
| `feature-new-website` | Feature branch used for the design revamp | Only for new features |
| `gh-pages` | Auto-generated by `npm run deploy` — powers the live site | **Never** |

### How the branches relate

```
main  ──(npm run deploy)──▶  gh-pages  ──▶  GitHub Pages  ──▶  Live website
 │
 └── feature-new-website  (merge into main when done)
```

### Rules to follow

- **Always make changes on `main`** (or a feature branch, then merge to `main`).
- **Never manually edit `gh-pages`.** It is always overwritten by `npm run deploy`.
- **Always run `npm run deploy` while on `main`.** Running it from any other branch will
  deploy the wrong files to the live site.

## 3. SSH Key Setup — Two GitHub Accounts

This laptop has two GitHub accounts configured with separate SSH keys so both can
be used simultaneously without conflicts.

### Accounts

| Account | Usage | SSH Key |
|---------|-------|---------|
| Corporate/work account | Default — all work/org repos | `~/.ssh/id_rsa` |
| `Itsmemukesh` (personal) | This portfolio repo | `~/.ssh/id_rsa_personal` |

### How it works — `~/.ssh/config`

The SSH config file routes each connection to the correct key based on a **host alias**:

```
# Corporate GitHub account (default)
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa

# Personal GitHub account (Itsmemukesh)
Host github-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa_personal
```

- Work repos use `git@github.com:org/repo.git` → picks up `id_rsa` automatically.
- This portfolio uses `git@github-personal:Itsmemukesh/portfolio.git` → picks up
  `id_rsa_personal` via the `github-personal` alias.

### Verifying both connections

```bash
ssh -T git@github.com          # Should print: Hi <work-username>!
ssh -T git@github-personal     # Should print: Hi Itsmemukesh!
```

### This repo's remote URL

```bash
git remote -v
# origin  git@github-personal:Itsmemukesh/portfolio.git (fetch)
# origin  git@github-personal:Itsmemukesh/portfolio.git (push)
```

The `github-personal` host alias (not `github.com`) ensures git uses the personal
SSH key for all push/pull operations on this repo.

> **If you ever re-clone this repo on a new machine**, set the remote with:
> ```bash
> git remote set-url origin git@github-personal:Itsmemukesh/portfolio.git
> ```

## 4. Running Locally

No build step is required — this is a plain static site.

```bash
npm start
```

This starts a local HTTP server at **http://localhost:3000** using Python's built-in
`http.server`. Open that URL in your browser to preview the site.

**Alternative** (if Python is not on PATH):

```bash
npx serve . -p 3000
```

## 5. Making and Committing Changes

### Editing content

| What to change | File |
|----------------|------|
| Page text, sections, structure | `index.html` |
| Colors, fonts, layout, spacing | `style.css` |
| Navigation behavior, animations | `app.js` |
| Profile photo | `assets/profile.jpg` |

### Commit workflow

```bash
# 1. Check what has changed
git status
git diff

# 2. Stage the changed files
git add index.html style.css app.js
# Or stage everything (excluding node_modules — covered by .gitignore)
git add .

# 3. Commit with a clear message
git commit -m "Brief description of what changed"

# 4. Push to GitHub
git push origin main
```

### Using a feature branch (for larger changes)

```bash
# Create and switch to a new feature branch
git checkout -b feature/my-new-section

# ... make your changes, then commit ...
git add .
git commit -m "Add new testimonials section"

# Push the feature branch
git push origin feature/my-new-section

# Merge back into main when ready
git checkout main
git merge feature/my-new-section
git push origin main

# Clean up the feature branch (optional)
git branch -d feature/my-new-section
git push origin --delete feature/my-new-section
```

## 6. Pushing Changes to GitHub

Since this repo uses the `github-personal` SSH alias, push commands work as normal:

```bash
git push origin main
```

Git will automatically use the personal SSH key (`id_rsa_personal`) because the remote
URL contains `github-personal` instead of `github.com`.

### If you get a permission denied error

1. Test the SSH connection first:
   ```bash
   ssh -T git@github-personal
   # Expected: Hi Itsmemukesh! You've successfully authenticated...
   ```

2. If the test fails, your SSH key may not be loaded. Add it:
   ```bash
   ssh-add ~/.ssh/id_rsa_personal
   ```

3. Verify the remote URL is using the alias (not `github.com`):
   ```bash
   git remote -v
   # Should show: git@github-personal:Itsmemukesh/portfolio.git
   ```

   If it shows `github.com` instead, fix it:
   ```bash
   git remote set-url origin git@github-personal:Itsmemukesh/portfolio.git
   ```

## 7. Deploying to the Live Website

Deployment pushes your local files to the `gh-pages` branch, which GitHub Pages
serves as the live website.

```bash
# IMPORTANT: always be on main before deploying
git checkout main

# Deploy
npm run deploy
```

Expected output:
```
> predeploy: "No build step needed for static site"
> deploy: gh-pages -d .
Published
```

The live site at **https://Itsmemukesh.github.io/portfolio** updates within 1–2 minutes.
If you still see the old version, do a hard refresh: `Ctrl + Shift + R`.

### What `npm run deploy` does internally

1. Takes all files in the current directory (`.`)
2. Creates a new commit on the `gh-pages` branch with those files
3. Force-pushes `gh-pages` to GitHub
4. GitHub Pages detects the push and rebuilds the live site

### GitHub Pages settings

GitHub Pages must be configured to serve from the `gh-pages` branch:

> GitHub repo → **Settings** → **Pages** → Source: `gh-pages` branch, `/ (root)`

```bash
# Step 8 — Wait 1-2 minutes, then verify at:

```

## 9. Troubleshooting

### `npm run deploy` gives "'gh-pages' is not recognized"

Windows is missing the `.cmd` wrapper for the `gh-pages` binary. Fix:

```bash
npm install
npm run deploy
```

### Site did not update after `npm run deploy`

**Most likely cause:** `npm run deploy` was run from the wrong branch.

Check which branch you were on and re-deploy from `main`:

```bash
git checkout main
npm run deploy
```

**Also try:** a hard refresh in the browser (`Ctrl + Shift + R`) since GitHub Pages
has a CDN cache that can serve stale content for a few minutes.

### `403 Permission denied` when pushing

The wrong GitHub account's credentials are being used. Verify:

```bash
ssh -T git@github-personal
# Must say: Hi Itsmemukesh!

git remote -v
# Must show: git@github-personal:Itsmemukesh/portfolio.git
```

If the remote shows `https://` instead of `git@`, switch it to SSH:

```bash
git remote set-url origin git@github-personal:Itsmemukesh/portfolio.git
```

### Work repos accidentally using personal SSH key (or vice versa)

Work repos use `git@github.com:...` and personal repos use `git@github-personal:...`.
As long as the remote URLs use the correct prefix, there is no conflict — SSH reads
the `~/.ssh/config` file and routes each connection to the right key automatically.
